# Context Pack Format

Use this reference when creating or reviewing a context pack.

## Required Sections

1. Task
2. Explicit Spec Facts
3. Inferences
4. Interface Contract
5. Similar Modules
6. Project APIs
7. Allowed Edit Paths
8. Forbidden Paths
9. Tests
10. Open Questions

## Fact Discipline

- Explicit facts come only from the Spec or direct code reads.
- Inferences must say what evidence produced them.
- Missing behavior belongs in Open Questions, not in implementation guesses.
- Similar modules are examples, not authority over the Spec.
